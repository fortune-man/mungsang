
## 기능 구현 목록

- 제한된 시간내 구현을 위해 임팩트 측정 후 우선순위 정렬
  구현 순서 정리
  1. 타이머 시작/정지 API 구현 → 타이머 저장 로직 → 기록 저장 테스트
  2. 랭킹 API 구현 → 정렬/집계 쿼리 작성 → 랭킹 결과 테스트
  3. 기본 UI 연결 (타이머 페이지) → API 연동 및 동작 테스트
  4. 간단한 배포 환경 설정 → 사용자 피드백 수집

### MVP 테스트 플로우
- API 테스트: Postman 또는 Swagger로 /start-timer, /stop-timer, /rankings 테스트.
- DB 확인: H2 Console로 데이터 저장 상태 확인.
- 전체 동작 통합 테스트: 타이머 시작 → 정지 → 랭킹 확인 플로우 테스트.

- 목적 의식 정의
   왜 이 프로젝트를 하려고 하는가?
* 개인적인 이유:
    * 조카를 비롯한 아동·청소년이 디지털 세상에서 건강한 습관을 가지도록 돕고 싶다.
    * 디지털 중독이나 역기능(소셜 미디어의 과도한 노출, 정신 건강 악화 등)에 대한 예방과 대처 방안을 제안하고 싶다.
    * 유튜브 a/b 테스트 결과 디지털 역기능보다 리프레쉬에 대한 수요 조금 더 확인되어 우선 '번아웃 단축'으로 주제를 개선
* 사회적 가치:
    * 디지털 환경에 쉽게 노출되는 직장인과 청소년들에게 리프레쉬와 재충전의 기회를 제공하는 서비스.
    * 사용자들에게 실제로 도움이 되는 습관 형성을 도와 사회에 긍정적인 영향을 미치는 도구를 개발하고 싶다.
* 개발적 이유:
    * 24시간 안에 최소 기능을 구현하며 빠르게 MVP를 검증하는 과정을 통해 개인 개발 역량을 향상.
    * Spring Boot와 웹 기술(JavaScript 등)을 활용하여 아이디어를 구체화하고, 실질적인 결과물을 만들어내는 경험 쌓기.
      목표 사용자
1. 조카와 같은 아동·청소년:
    * 디지털 세계의 부정적 영향에서 보호받고, 올바른 디지털 습관을 형성하도록 돕는 대상.
2. 번아웃에 노출된 직장인:
    * 간단한 리프레쉬 타이머 기능으로 스트레스를 줄이고, 지속 가능한 작업 루틴을 만드려는 대상.
      결과로 얻고자 하는 것
* 서비스가 실제로 아동·청소년과 직장인의 문제를 해결하는 데 효과적인지 검증.
* 최소한의 기능(MVP)으로도 가치를 제공할 수 있는지 실험.
* 프로젝트 결과물을 포트폴리오로 활용해 개발자로서 성장.

2. 구체적 목표 설정
   1단계: 24시간 안에 달성할 목표
1. 핵심 기능 정의 및 구현:
    * 멍 때리기 타이머 기능 (백엔드 API + 간단한 프론트엔드).
    * 기록 저장 (Spring Boot + H2 Database).
2. 프로젝트 환경 설정 및 초기화:
    * Spring Boot 설정 완료.
    * README 작성 및 첫 커밋 완료.
3. 실행 가능 여부 검증:
    * 브라우저에서 기본적인 동작 확인(타이머 작동 및 기록 저장).
      2단계: MVP 실험 (추가 목표)
* 사용자 랭킹 기능 구현.
* 기본 UI를 개선하여 사용자 경험(UX) 향상.
* 사용자 기록을 바탕으로 긍정적 피드백 제공(상장 발급 등).

3. 목적을 기반으로 한 행동 계획
   핵심 질문
* "이 프로젝트가 사용자의 문제를 해결하는 데 효과적인가?" -> 이를 검증하기 위해 **최소한의 기능(MVP)**으로 시작.

### 프로젝트 구조

src/
├── main/
│   ├── java/com/example/timerapp/
│   │   ├── controller/       // API 엔드포인트
│   │   ├── service/          // 비즈니스 로직
│   │   ├── repository/       // 데이터베이스 인터페이스
│   │   ├── entity/           // 데이터베이스 엔티티 정의
│   │   ├── dto/              // 데이터 전송 객체 (클라이언트 ↔ 서버 간 데이터 교환)
│   │   ├── config/           // 설정 (필요 시)
│   │   └── TimerAppApplication.java  // 메인 애플리케이션 클래스
│   ├── resources/
│   │   ├── application.properties  // 데이터베이스 및 환경 설정
│   │   └── templates/       // 프론트엔드 HTML 템플릿 (Thymeleaf 사용 시)
│   └── static/              // 정적 리소스 (CSS, JS 등)
└── test/
├── java/com/example/timerapp/  // 테스트 코드
